# EVM 和非 EVM，谁更有前途？

## EVM 是什么？

EVM 即以太坊虚拟机，简单理解就是一个通用的执行环境。没有它，以太坊的智能合约的确定性将无法保障。只要支持智能合约功能，虚拟机在当前的技术路线下就不可缺少。

## EVM 有何问题？

- 不支持浮点数及缺少标准库。
- 合约代码不支持优化升级，只支持重新部署。

## 兼容 EVM

### 优势

- 用户迁移成本低。
- 继承现有的 dApps。
- 技术成熟度更高。
- 更容易与以太坊生态系统实现互操作性，即网络效应。

### 劣势

- 资本流转很容易，因为资本的转换成本很低。
- 受到以太坊路线图的约束，这可能会影响发布计划。
- 创新受到了 EVM 规则的约束。

## 不兼容 EVM

### 优势

- 实现差异化功能。
- 对既有用户产生“锁住效应”。
- 独立于以太坊路线图。

### 劣势

- 开发者的进入障碍：新的语言 / 框架 / 开发环境。
- 用户的进入障碍：新的钱包 / 接口 / 应用。
- 从零开始增长用户基数。
- 由于处于孤岛状态，网络效应降低 - 跨链 DeFi 可组合性。

## 兼容 EVM 的公链竞争

- BSC
- Polygon
- Arbitrum / Optimism
- Avalanche

## 不兼容 EVM 的公链竞争

- Solana
